## BOJ1629 - 곱셈  
(문제: https://www.acmicpc.net/problem/1629)  
(시간: 52분)  

거듭제곱 문제. 그런데 밑과 지수가 다 21억인..  
따라서 A를 21억번 곱해줄수도 없는 노릇이고, 이럴땐 ^2을 여러번 해주면 된다.  

b가 짝수일때, b = 2k &nbsp; &nbsp; &nbsp; → &nbsp; &nbsp; a^b^ = (a^k^)^2^  
b가 홀수일때, b = 2k+1 &nbsp; → &nbsp; &nbsp; a^b^ = a(a^k^)^2^  

재귀문제를 재귀로 구현할 수도 있고 스택 자료구조로 구현할 수도 있는데(함수 = 반복문+스택), 이 문제같은 경우는 스택이 좀 더 이해하기 쉬운듯하다(문제마다 스택이 더 직관적인지, 재귀가 더 직관적인지가 다른듯).  

재귀함수가 코드는 간결하지만 경우에 따라 논리가 헷갈릴 수 있고, 특히 스택 메모리가 1mb로 제한된 SWEA 같은데서는 함수를 재귀로 호출했을 때 스택오버플로 날 수 있으니까(4000번 정도 호출하면 남), 그럴 때는 비재귀로 풀어서 스택을 힙영역에 두고 쓰는게 좋을 것 같다. 게다가 경우에 따라선 함수 콜연산 때문에 시간초과도 날 수 있다. 물론 이 문제는 최대 31번 호출하기 때문에 둘 다 가능.
